## 📝 如何使用脚本

### 1. 设置环境变量（每次使用前）

```bash
# 后端部署
export EC2_HOST=ec2-user@YOUR_BACKEND_IP
export SSH_KEY=./your-key.pem

# 前端部署
export FRONTEND_EC2_HOST=ubuntu@YOUR_FRONTEND_IP  
export BACKEND_API_URL=http://YOUR_BACKEND_IP:8080/api
export SSH_KEY=./your-key.pem
```

### 2. 运行部署脚本

```bash
# 后端
./deploy.sh

# 前端
./deploy-frontend.sh
```

## ⚠️ 重要提醒

1. **永远不要**将 `.pem`、`aws-config.env`、密码文件提交到 Git
2. **永远不要**在文档中硬编码真实的 IP 地址、密码、密钥
3. 使用 **环境变量** 或 **GitHub Secrets** 来管理敏感信息
4. 定期检查 `git status` 确保敏感文件未被追踪

## 🔍 检查是否有敏感信息泄露

```bash
# 检查是否有未被 ignore 的敏感文件
git status

# 搜索可能的 IP 地址
grep -r "\.\.\.\..*" deployment/ --exclude-dir=node_modules

# 搜索可能的密码
grep -ri "password" deployment/ --exclude=*.md
```
